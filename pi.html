What it is

A second brain
that never sleeps

A personal AI runs best when it's always there — to take a capture from your phone at midnight, to run the automations on time, to guarantee tonight's snapshot even if the laptop is shut. So the vault now lives on a Raspberry Pi 5, on around the clock, doing all of that on its own. The laptop became the workbench; the Pi became the brain. Nothing about the vault depends on any one machine being awake.

The mental model

Two machines,
one clean split

Brain and workbench, deliberately kept apart — and held identical by a continuous two-way mirror. Either one can fail and the vault survives on the other.

Capabilities

What "always-on"
actually buys

Runs 24/7 on a Pi 5
An 8GB Raspberry Pi 5 on Debian, on around the clock. VaultChat runs as a systemd service that starts on boot and restarts on failure — the front door is open even at 3am.
systemd · enabled on boot
Local RAG on the ARM CPU
The whole retrieval engine — index and embeddings — runs natively on the Pi's CPU. No GPU, no cloud inference. Search and the you-first agent answer entirely on-device.
qmd · CPU embeddings
Automations on Cron
The Windows Task Scheduler jobs became plain cron: a nightly git snapshot at 00:00 and the AI morning digest at 07:00 — on the Pi's own clock, whether or not the laptop is awake.
00:00 commit · 07:00 digest
Two-Way Live Mirror
Syncthing keeps the Pi and the laptop identical — peer-to-peer, no cloud middleman. Edit on either side and the change lands on the other in seconds, deletes included. Verified both directions.
Syncthing · seconds
Backed Up on Both
Sync is not backup — a delete propagates. So each machine also keeps its own independent git history. Two snapshots of every night, on two disks: lose either machine and nothing is gone.
dual git history
Private Remote Access
The Pi joins a Tailscale tunnel only your own devices share, so the phone reaches VaultChat from anywhere — coffee shop, campus, abroad — with no port forwarding and no public door to the internet.
Tailscale · MagicDNS
Guaranteed Nightly Snapshot
Because the Pi is always on, the midnight commit can't be missed by a closed lid. The first fully autonomous snapshot landed at 00:00:01 with no human in the loop — and every night since.
never a missed night
Self-Reporting Automations
Every job logs whether it ran or failed to a durable file and surfaces it in the morning digest with a staleness check. If the Pi ever misses a beat, you find out the next morning — not by accident.
observability built in
Standalone & Portable
A whole personal AI OS in a box: clone the repo, point the config, run. Interim on the SD card today; the heavy corpus moves onto an NVMe disk in the next phase. The vault outlives the hardware.
clone · configure · run
