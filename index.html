System Design

Two doors,
one second brain

The me-layer is the front door — you talk to your personal context first. From there it operates in two modes: ASK (qmd retrieves relevant chunks from knowledge wikis, fast and cheap) or MAINTAIN (full wiki schema loads only when editing). Knowledge wikis stay isolated; they serve you, they don't address you. And there are now two ways in: the desktop Claude Code session, and VaultChat on your phone — the same me-layer, a second entryway.

01
Always-On — Self-Hosted on a Pi
The whole second brain now lives on a Raspberry Pi 5 running 24/7 — VaultChat, local RAG, and the nightly automations all run on-device, no laptop required. A two-way Syncthing mirror keeps the Pi and the laptop bit-for-bit identical in seconds, while each machine keeps its own git history for mutual backup. The schedulers became plain cron; the phone reaches it anywhere over Tailscale. A standalone personal AI OS in a box — brain on the Pi, workbench on the laptop.

VaultChat — Phone Front Door
A second front door — the vault on your phone. 03
Me-Layer Intelligence
The personal wiki is the front door. It holds your logs, obligations, and how-you-function model. You address it first; knowledge wikis are what it reaches into. One fixed rule: suggest, recommend, push back — never decide for you.

04
Local RAG Search
3-stage retrieval funnel: BM25 keyword + vector semantic hybrid. 1,800+ vectors across 3 collections, embedded on-device on the Pi's CPU. Section Map targeting avoids reading full pages.
qmd engine
05
Automated Catalog Maintenance
Two Python hooks fire automatically. A PostToolUse sentinel tracks every wiki edit as it happens. At session end, a Stop hook writes log.md from staged metadata — pages created, pages updated, source. Stats and vault diagnostics refresh async. Zero tokens, zero manual steps.
stop hook pipeline
06
Opinionated Knowledge Structure
Strict concept page templates enforce Intuition → Key Idea → Derivation → Worked Example. Auto-file vs. propose vs. never rules. Methodology stances resist silent drift.
schema-enforced

Multi-Model Orchestration
Intelligent task-to-model routing. Haiku for log appends and file moves, Sonnet for standard ingest, Opus for cross-page synthesis and race prep. Minimum tokens, adequate output.

Custom Skills & Slash Commands
/hw-concept-map maps problem sets to concept pages. /proj-concept-map builds structured trackers. /marp generates slide decks from wiki content. /notebooklm integrates external sourcing. /diagnose, /grill-me, /handoff, and /zoom-out cover the meta-cognitive layer.

09
Daily AI Morning Brief
Automated email digest fires at 7am IST via cron on the always-on Pi. Gemini 3.5 Flash with Google Search grounding covers 9 sections — investment, compute, hardware, research, sports science — filtered through your personal profile. 19× cheaper than Claude web_search: $0.031/day vs $0.59.
Gemini · Google Search
10
Nightly Auto-Commit
Cron on the always-on Pi fires at midnight. Stages all tracked and untracked changes, exits cleanly if nothing has changed, otherwise snapshots with a dated commit. Status written to a JSON file — the morning digest confirms whether it ran. The vault is always git-safe without a single manual step.
Pi cron · git
11
Living Memory System
Two-mode memory that evolves with use — no configuration required. The moment you signal something is worth keeping (confirm an approach, say you want a workflow reproducible), it writes automatically and surfaces the write in one line. When it infers a pattern you didn't flag, it asks first. The signal is always yours; the write is automatic on your cue.

Bounded at 20 active entries. When capacity is reached, stale memories are archived into memory/archive/ — never deleted, just deprioritised. The full memory store lives physically inside the Vault via a symlink: visible in any editor, version-controlled, portable. Clone the repo to a new machine, create one symlink, memory travels with you.
